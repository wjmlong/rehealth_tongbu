## Key gotchas (for future runs)
- **WSL `/mnt/d` is extremely slow for bulk git ops.** `git add -A` over ~6000 files took 15+ min (process was in `Sl+` sleeping on I/O, not computing). Killed it; did a lean commit staging only the 5 LFS files + supporting edits (`git reset -q` then targeted `git add -f`). The 1.37 GB LFS hashing was also slow (~5 min) but unavoidable.
- **Avoid inline `$PATH` in `wsl -e bash -lc "..."`** — Git Bash expands `$PATH` to the Windows PATH and breaks commands (saw `tail: command not found`, EXIT=127). Fix: write a script file and run `wsl bash /mnt/d/.../script.sh` (no inline $PATH). This pattern worked reliably all session.
- A stale `.git/index.lock` from a killed `git add -A` blocked later `git add -f`; `rm -f .git/index.lock` (after confirming no live git process) resolved it. Killing mid-`git add -A` is safe — git writes to index.lock and only renames atomically at the end, so the real index stays intact.
