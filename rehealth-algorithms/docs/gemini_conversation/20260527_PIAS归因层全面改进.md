## 二、Bug 修复细节

### 2.1 risk_scorer.py — model_auc 硬编码

**问题**: 第75行 `"model_auc": 0.847` 写死，实际 V2 模型 AUC 为 0.767。

**修复**: `__init__` 新增可选 `model_auc` 参数，默认 None 时返回 `"not_evaluated"`。`ModelRegistry.register()` 同步支持传入。

```python
# Before
"model_auc": 0.847,

# After
def __init__(self, model_path: str, model_auc: float | None = None):
    ...
    self._model_auc = model_auc

# In predict():
"model_auc": self._model_auc if self._model_auc is not None else "not_evaluated",
```

### 2.2 inference.py — memory_snapshot 误传

**问题**: 第45行 `scorer.predict(req.memory_snapshot)` — memory_snapshot 是任意 dict（含 age_bracket、bmi_level 等脱敏字段），不含 scorer 需要的 16 个 FEATURE_COLS，运行时 KeyError。

**修复**: 先从 memory_snapshot 提取 FEATURE_COLS 子集，缺字段返回 400。

```python
snap = req.memory_snapshot or {}
missing = [c for c in scorer.FEATURE_COLS if c not in snap]
if missing:
    raise HTTPException(400, f"memory_snapshot missing required features: {missing}")
features = {c: snap[c] for c in scorer.FEATURE_COLS}
risk_result = scorer.predict(features)
```

### 2.3 attribution.py — hash() 编码不稳定

**问题**: `hash(snap.get("bmi_level", "")) % 4` — Python `hash()` 在不同进程/版本间结果不一致（PYTHONHASHSEED 随机化），同一个 "normal" 在不同启动中可能映射到不同整数。另外 `gender_encoded` 错用了 `age_bracket % 2`。

**修复**: 显式字典映射，未知值 fallback 到 0。

```python
BMI_MAP = {"underweight": 0, "normal": 1, "overweight": 2, "obese": 3}
BP_MAP = {"normal": 0, "elevated": 1, "stage1": 2, "stage2": 3, "crisis": 4}
ACTIVITY_MAP = {"sedentary": 0, "light": 1, "moderate": 2, "vigorous": 3}
GENDER_MAP = {"male": 0, "female": 1}
```

## 三、Level 1 个体预测改进 (individual_prediction.py)

### 3.1 框架概述

IndividualRiskPredictor 负责对单个用户的每日 CVD 风险评分序列进行趋势分析和预测。

**输入**: `risk_history` — 每天一条 `{date, Y(风险分), Z(是否干预日)}`  
**输出**: 30天预测曲线（维持现状 vs 执行计划）、置信区间、回顾性 ATT、App文案

### 3.2 改进 A: 指数衰减预测替代线性外推

**旧方法**: `forecast = last_Y + slope * t` — 线性外推可能突破 [0,1] 边界，且不符合风险变化的生物学规律（变化速率应随偏离均值而衰减）。

**新方法**:
1. 对 Y 取 log: `log_Y = log(clip(Y, 0.01, 0.99))`
2. 对 `(t, log_Y)` 做加权线性回归得到 `rate`
3. 预测: `forecast = last_Y * exp(rate * t)`
4. 指数模型天然收敛，风险不会线性无限增长

```python
Y_safe = np.clip(Y, 0.01, 0.99)
log_Y = np.log(Y_safe)
rate_all, log_intercept_all = self._weighted_fit(t, log_Y, weights)
forecast_no_action = np.clip(last_Y * np.exp(rate_all * future_t), 0, 1)
```

### 3.3 改进 B: 预测置信区间

基于加权回归残差估计预测不确定性:

```python
residuals = log_Y - (log_intercept_all + rate_all * t)
sigma = sqrt(sum(weights * residuals²) / sum(weights))
prediction_se = sigma * sqrt(1 + 1/N)
ci_multiplier = 1.96 * prediction_se

forecast_ci_upper = clip(last_Y * exp(rate * t + ci_multiplier), 0, 1)
forecast_ci_lower = clip(last_Y * exp(rate * t - ci_multiplier), 0, 1)
```

返回中新增 `forecast_ci_upper`、`forecast_ci_lower` 各30个值，报告 metrics 新增 `confidence_band_width`。

### 3.4 改进 C: 移除 magic number 0.7

**旧方法**: 干预数据不足7天时 `slope_plan = slope_all * 0.7`（假设干预比不干预好30%，无依据）。

**新方法**: `rate_plan = rate_all`（保守假设：数据不足则不做优劣判断），并标记 `intervention_data_sufficient: False`。报告文案变为"干预数据不足，暂以整体趋势预测"。

### 3.5 改进 D: 配对 ATT

**旧方法**: `ATT = mean(Y[Z=1]) - mean(Y[Z=0])` — 简单均值差有时间混淆（干预日可能集中在后期，此时风险已因其他原因变化）。

**新方法**: 按周配对 — 同一周内干预日均值 vs 非干预日均值，取配对差的均值。消除周际趋势干扰。

```python
for w in range(n_weeks):
    week_Y, week_Z = Y[w*7:(w+1)*7], Z[w*7:(w+1)*7]
    if both intervention and control days exist in this week:
        paired_diffs.append(mean(Y[Z=1]) - mean(Y[Z=0]))
return mean(paired_diffs) if >= 2 paired weeks else fallback to simple diff
```

## 四、Level 2 群体归因改进 (group_attribution.py)

### 4.1 框架概述

GroupAttributionEngine 负责在群体层面评估健康干预的因果效应，用于保险结算。

**输入**: `user_records` — 每用户一条 `{device_id, Z(干预/对照), delta_Y(风险变化), features(协变量)}`  
**输出**: ATT(平均处理效应)、95%CI、PSM平衡诊断、Rosenbaum敏感性Γ、结算报告

**方法流程**: 倾向得分估计 → PSM匹配 → 双重稳健估计(DRE) → Bootstrap CI → Rosenbaum敏感性分析

### 4.2 改进 A: sklearn LogisticRegression

**旧方法**: 手写 L2 正则化逻辑回归 — 自实现 sigmoid、负对数似然、梯度、scipy L-BFGS-B 优化。约30行代码。

**新方法**: 
```python
from sklearn.linear_model import LogisticRegression
lr = LogisticRegression(C=10, solver="lbfgs", max_iter=300, random_state=42)
lr.fit(X, Z)
ps = lr.predict_proba(X)[:, 1]
return np.clip(ps, 0.05, 0.95)
```

C=10 对应旧代码 λ=0.1 (C=1/λ)。效果等价，代码从30行降到5行，且经过大量单元测试验证的工业实现。

### 4.3 改进 B: KD-tree PSM 匹配

**旧方法**: O(n²) 双重循环 — 对每个 treated，遍历所有 control 找最近邻。200人 = 10,000次比较。

**新方法**:
```python
from sklearn.neighbors import NearestNeighbors
nn = NearestNeighbors(n_neighbors=1, metric="euclidean")
nn.fit(ps_control.reshape(-1, 1))
distances, indices = nn.kneighbors(ps_treated.reshape(-1, 1))
```

O(n log n) 建树 + O(n log n) 查询。按距离从小到大排序后无放回匹配（优先匹配最近的对），caliper 外的自动过滤。

### 4.4 改进 C: 自适应 caliper

**旧方法**: 固定 `caliper = 0.2` — 不考虑倾向分的实际分布。若 PS 分布很窄，0.2 太宽（几乎不过滤）；若很宽，0.2 太严（丢掉很多配对）。

**新方法**: 默认 `caliper = "auto"`，运行时计算:
```python
logit_ps = log(ps / (1 - ps))
caliper_value = 0.2 * std(logit_ps)
```

这是 Austin (2011) 推荐的最优 caliper 宽度，在 MSE 和偏差之间取得平衡。config 中仍可传固定数值覆盖。

### 4.5 改进 D: Rosenbaum 敏感性分析

**动机**: PSM+DRE 假设"无未观测混杂"（所有影响干预分配的因素都被纳入协变量）。现实中这个假设很难满足。Rosenbaum bounds 回答："如果存在未观测混杂，结论需要多大的混杂才会被推翻？"

**方法**:
- Γ = 1.0 表示无混杂（完美随机化）
- Γ = 2.0 表示未观测混杂使干预概率翻倍
- 在多个 Γ 值 [1.0, 1.25, 1.5, 1.75, 2.0, 2.5, 3.0] 下计算 Wilcoxon signed-rank 检验的 p 值上界
- 找到使 p > 0.05 的最小 Γ → `gamma_sensitivity`

```python
gamma_values = [1.0, 1.25, 1.5, 1.75, 2.0, 2.5, 3.0]
for gamma in gamma_values:
    adjusted = diffs * gamma
    _, p_value = wilcoxon(adjusted, alternative="less")
    if p_value < 0.05:
        max_robust_gamma = gamma
    else:
        break
```

**解读**:
- Γ ≥ 2.5: "结论非常稳健，即使存在较大未观测混杂也不影响结论"
- Γ ≥ 1.5: "结论较稳健，可容忍中等程度的未观测混杂"
- Γ > 1.0: "结论略敏感，小幅未观测混杂可能改变结论"
- Γ = 1.0: "结论对未观测混杂敏感，建议谨慎解读"

### 4.6 改进 E: 结算报告增加 Γ

`_render_settlement_report` 的 method 段追加敏感性分析描述，metrics 新增 `gamma_sensitivity` 字段。保险方可据此评估结论可靠性。
