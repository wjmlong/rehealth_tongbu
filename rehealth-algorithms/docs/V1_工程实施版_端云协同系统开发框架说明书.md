
**V1 与初代的核心区别**：

| 维度 | 初代 PHM (V0) | V1 工程实施版 |
|------|---------------|---------------|
| 大模型位置 | 手机端侧（Gemma 4 E4B 量化部署） | 云端 LLM API（Claude/GPT/OpenRouter，后期可迁移自建） |
| 个性化机制 | 端侧 LoRA 增量训练 + EWC 防遗忘 | **端侧高密度记忆文件** + 云端 RAG 注入 |
| 风险评分 | 端侧 ONNX 模型推理 | **云端 CatBoost**（已训练 AUC=0.847, SHAP可解释，可扩展多模型） |
| 自然语言干预 | 端侧 Gemma 推理 | **云端 LLM API 推理**（Claude/GPT，零运维按量付费） |
| 归因引擎 | 端侧 PSM + Bootstrap | **云端双重稳健估计（DRE）** + Bootstrap |
| 防篡改签名 | 端侧 HMAC（Secure Enclave） | **云端 Ed25519 非对称签名**（不可抵赖） |
| 设备要求 | 12GB+ RAM | **任意机型**（端侧仅跑统计算子） |

**为什么做这个调整**：不是初代方案不好——初代方案是专利壁垒和长期愿景的技术资产。但在当前阶段（种子轮资金刚到位，需要 6-8 个月内跑出真实数据），V1 方案能把工程风险降到最低：
- 不需要解决端侧 LoRA 训练的发热/OOM/灾难性遗忘问题
- 不需要 12GB RAM 设备门槛（用户池不受限）
- 归因签名用非对称密钥，保司法务可直接签字

### 3.2 夜间做梦机制（V1 简化版）

V1 的做梦机制**只做统计学特征聚合**，不做任何模型训练（区别于初代的 LoRA 微调 + EWC 防遗忘）。

```
触发条件: [充电] AND [WiFi] AND [锁屏] AND [凌晨 02:00-05:00]
执行引擎: 纯 CPU (C++ / Python), 无 GPU 调用
预计耗时: 10-30 秒 (vs. 初代的 8-15 分钟)
内存占用: < 50MB (vs. 初代的 2GB+)

任务清单:
  Task 1: 从 SQLite 读取过去 24h 原始手表数据          (~5ms)
  Task 2: 14天滑动窗口 → mean/std/slope/分级            (~10ms)
  Task 3: 夜间血压杓型判定 (dipper/non_dipper/reverse)   (~5ms)
  Task 4: 干预依从性统计 (运动触发次数/饮食执行率)       (~5ms)
  Task 5: 更新 user_health_memory.json                  (~5ms)
  Task 6: 追加 [ts, Z, Y, confounders] 到上传队列       (~5ms)
```

### 3.3 16 维特征提取算子

```python
# bodyup_edge_sdk/core/feature_extractor.py

import numpy as np
from datetime import datetime
import math

class HealthFeatureExtractor:
    """16维生理特征提取 · 纯CPU · 兼容所有机型"""

    def __init__(self, window_days: int = 14):
        self.window_days = window_days

    def extract_bp_features(self, sbp_hourly: list, dbp_hourly: list) -> dict:
        sbp = np.array(sbp_hourly[-(self.window_days * 24):])
        dbp = np.array(dbp_hourly[-(self.window_days * 24):])

        # 夜间杓型判定
        daily = sbp.reshape(-1, 24)
        night = daily[:, 0:6].mean(axis=1)
        day = daily[:, 10:22].mean(axis=1)
        dip = (day.mean() - night.mean()) / day.mean()

        pattern = ("extreme_dipper" if dip >= 0.20 else
                   "dipper" if dip >= 0.10 else
                   "non_dipper" if dip >= 0.0 else
                   "reverse_dipper")

        slope = np.polyfit(np.arange(len(sbp)), sbp, 1)[0] if len(sbp) > 1 else 0.0

        return {
            "sbp_mean": round(float(np.mean(sbp)), 1),
            "sbp_std": round(float(np.std(sbp)), 1),
            "sbp_slope": round(float(slope), 4),
            "dbp_mean": round(float(np.mean(dbp)), 1),
            "dbp_std": round(float(np.std(dbp)), 1),
            "night_bp_pattern": pattern
        }

    def extract_cardiac_features(self, hr_series: list, hrv_series: list) -> dict:
        hr = np.array(hr_series[-(self.window_days * 24):])
        hrv = np.array(hrv_series[-self.window_days:])
        return {
            "resting_hr_mean": round(float(np.mean(hr)), 1),
            "hrv_rmssd_mean": round(float(np.mean(hrv)), 1)
        }

    def extract_activity_features(self, calories: list, sport_events: list,
                                  sleep_hours: list) -> dict:
        cal = np.array(calories[-self.window_days:])
        sleep = np.array(sleep_hours[-self.window_days:])
        trend = np.polyfit(np.arange(len(cal)), cal, 1)[0] if len(cal) > 1 else 0.0
        return {
            "active_calories_daily_mean": round(float(np.mean(cal)), 1),
            "active_calories_trend": round(float(trend), 4),
            "sport_freq_weekly": round(len(sport_events) / max(self.window_days / 7, 1), 1),
            "sleep_hours_mean": round(float(np.mean(sleep)), 1)
        }

    def encode_time_confounders(self) -> dict:
        now = datetime.now()
        angle = 2 * math.pi * now.timetuple().tm_yday / 365.25
        return {
            "season_sin": round(math.sin(angle), 4),
            "season_cos": round(math.cos(angle), 4),
            "day_of_week": now.weekday(),
            "is_weekend": 1 if now.weekday() >= 5 else 0
        }
```

```python
# bodyup_edge_sdk/core/desensitizer.py

class HealthDesensitizer:
    """将原始记忆文件脱敏为不可逆的统计向量"""

    AGE_BRACKETS = [(0,30,1), (30,40,2), (40,50,3), (50,60,4), (60,70,5), (70,120,6)]
    BMI_LEVELS = [(0,18.5,"underweight"), (18.5,24,"normal"),
                  (24,28,"overweight"), (28,100,"obese")]
    ACTIVITY_LEVELS = [(0,100,"sedentary"), (100,200,"low"),
                       (200,350,"moderate"), (350,9999,"active")]

    def desensitize(self, memory: dict) -> dict:
        static = memory["static_baseline"]
        dyn = memory["dynamic_memory_vectors"]

        age = static["age"]
        age_bracket = next(b for lo,hi,b in self.AGE_BRACKETS if lo <= age < hi)
        bmi = static["bmi"]
        bmi_level = next(l for lo,hi,l in self.BMI_LEVELS if lo <= bmi < hi)
        cal = dyn["active_calories_daily_mean"]
        act_level = next(l for lo,hi,l in self.ACTIVITY_LEVELS if lo <= cal < hi)

        return {
            "device_id": memory["device_id"],
            "disease_type": memory["disease_type"],
            "memory_snapshot": {
                "age_bracket": age_bracket,
                "bmi_level": bmi_level,
                "bp_variability": dyn["sbp_std"],
                "night_bp_pattern": dyn["night_bp_pattern"],
                "activity_level": act_level,
                "sleep_quality_index": dyn.get("sleep_quality_index", 0.5),
                "resting_hr_mean": dyn["resting_hr_mean"],
                "hrv_rmssd_mean": dyn["hrv_rmssd_mean"],
                "intervention_compliance": memory["intervention_compliance"]["diet_compliance_rate"]
            },
            "attribution_timeseries": memory["risk_score_history"]
```

## 4. 云端设计详解

### 4.1 双轨推理：CatBoost 评分 + LLM API 处方

V1 的关键设计决策：**风险评分和干预处方分由两个模型负责**。

```
为什么不用 LLM 直接做风险评分？

1. 16维数据是强结构化数值, LLM 处理数值时存在"数值幻觉"
   → LLM 不真正理解 145 > 140 > 120 的物理含义
2. 保险精算师需要可解释性 (SHAP 特征贡献)
   → CatBoost 天然支持, LLM 做不到
3. 推理成本: CatBoost 毫秒级 vs. LLM 秒级
   → 高并发场景成本差 1000 倍

但 LLM 在以下环节不可替代：
  → 将风险分值翻译为用户能读懂的自然语言干预建议
  → 结合用户记忆快照做个性化表达
  → 未来接入对话式健康陪伴 (AInudge)

为什么 LLM 用 API 而不是自建？
  → 种子轮阶段，自建 Gemma+vLLM 需要 GPU 服务器 (~¥8000/月) + ML 运维
  → 100 用户每天 1 次调用: API 费用 ~$9/月 vs 自建 ~$1200/月
  → 没有自有医疗对话数据可以微调，用 Claude/GPT 开箱即用
  → 上传的是脱敏向量，不是原始健康数据，隐私无忧
  → 等用户规模破万 + 积累真实交互数据后，再迁移自建 (V1.5)
```

#### CatBoost 风险评分模型

已基于 NHANES 2015-2016 训练完成，AUC=0.847。模型文件: `rehealth_v2_final.pkl`。

```python
# bodyup_cloud_actuary/engine/risk_scorer.py

import joblib
import shap
import numpy as np
import pandas as pd
from typing import Dict

class CVDRiskScorer:
    """
    心脑血管风险评分器 — 基于已训练的 CatBoost 模型
    模型: rehealth_v2_final.pkl (AUC=0.847, NHANES 2015-2016)
    输入: 16字段 (对齐官网表单 + 手表动态更新)
    输出: 风险分值 Y ∈ [0,1] + SHAP 特征贡献
    """

    FEATURE_COLS = [
        "age", "gender", "bmi", "sbp", "dbp",
        "fasting_glucose", "total_cholesterol", "ldl", "hdl", "triglycerides",
        "exercise_days", "smoking", "drinking",
        "diabetes_history", "hypertension_history", "family_history"
    ]

    CAT_FEATURES = [
        "gender", "smoking", "drinking",
        "diabetes_history", "hypertension_history", "family_history"
    ]

    def __init__(self, model_path: str):
        self.model = joblib.load(model_path)
        self.explainer = shap.TreeExplainer(self.model)

    def predict(self, features: dict) -> dict:
        x = pd.DataFrame([{f: features.get(f, 0) for f in self.FEATURE_COLS}])
        for col in self.CAT_FEATURES:
            x[col] = x[col].astype(int)

        risk_score = float(self.model.predict_proba(x)[0, 1])
        shap_values = self.explainer.shap_values(x)[0]

        contributions = {
            self.FEATURE_COLS[i]: round(float(shap_values[i]), 4)
            for i in range(len(self.FEATURE_COLS))
        }

        return {
            "risk_score": round(risk_score, 4),
            "risk_level": self._grade(risk_score),
            "feature_contributions": contributions,
            "model_version": "v2.0",
            "model_auc": 0.847
        }

    @staticmethod
    def _grade(score: float) -> str:
        if score < 0.3: return "low"
        if score < 0.5: return "moderate"
        if score < 0.7: return "high"
        return "very_high"


class ModelRegistry:
    """
    多模型注册表 — 方便后续接入新数据源或升级模型
    当前: rehealth_v2_final.pkl (CatBoost, NHANES)
    未来: 可挂载其他梯度提升模型 (LightGBM 等) 或新版 CatBoost, 不改上层代码
    """

    def __init__(self):
        self._models: Dict[str, CVDRiskScorer] = {}

    def register(self, name: str, model_path: str):
```

#### 4.2.1 Level 1 · 个人预测性归因（面向用户的 App 卖点）

用户打开 App 看到的不是枯燥的统计数字，而是**两条风险走势线**：
- 🔴 "维持现状"：如果你接下来什么都不做，风险分会怎么变
- 🟢 "执行计划"：如果你跟着干预方案走，风险分会怎么变

这是**预测性**的（前瞻），不是纯回溯归因。底层逻辑：
1. CatBoost 每天算出一个风险分 Y
2. 积累 14+ 天的 Y 序列后，用简单时序模型（加权线性回归）外推未来 30 天
3. "执行计划"那条线基于干预期的改善斜率估算

```python
# bodyup_cloud_actuary/engine/individual_prediction.py

import numpy as np
from typing import List, Dict

class IndividualRiskPredictor:
    """
    单用户风险趋势预测 — App 产品核心体验
    输出: "维持现状"和"执行计划"两条未来30天走势线
    """

    def __init__(self, config: dict):
        self.min_history_days = config.get("min_history_days", 14)
        self.forecast_days = config.get("forecast_days", 30)
        self.decay_factor = config.get("decay_factor", 0.95)

    def predict(self, risk_history: List[Dict]) -> dict:
        """
        risk_history: [{date, Y, Z}, ...] 按日期升序
        Y: 当日风险分 (CatBoost 输出)
        Z: 1=干预日, 0=非干预日
        """
        if len(risk_history) < self.min_history_days:
            return {"status": "accumulating",
                    "days_available": len(risk_history),
                    "days_needed": self.min_history_days}

        dates = np.arange(len(risk_history), dtype=float)
        Y = np.array([r["Y"] for r in risk_history])
        Z = np.array([r["Z"] for r in risk_history])

        # 指数衰减权重 (近期数据权重更高)
        weights = self.decay_factor ** np.arange(len(Y) - 1, -1, -1)

        # "维持现状"趋势: 用全部数据的加权线性回归
        slope_all, intercept_all = self._weighted_fit(dates, Y, weights)

        # "执行计划"趋势: 用干预期数据的斜率
        intervention_mask = Z == 1
        if intervention_mask.sum() >= 7:
            slope_plan, _ = self._weighted_fit(
                dates[intervention_mask], Y[intervention_mask],
                weights[intervention_mask])
        else:
            slope_plan = slope_all * 0.7  # 干预数据不足时保守估计

        # 外推未来
        last_Y = float(Y[-1])
        last_t = float(dates[-1])
        future_t = np.arange(1, self.forecast_days + 1)

        forecast_status_quo = np.clip(
            last_Y + slope_all * future_t, 0, 1).tolist()
        forecast_with_plan = np.clip(
            last_Y + slope_plan * future_t, 0, 1).tolist()

        # 回溯性 ATT (辅助数据)
        if intervention_mask.sum() >= 7 and (~intervention_mask).sum() >= 7:
            att = float(Y[intervention_mask].mean() - Y[~intervention_mask].mean())
        else:
            att = None

        return {
            "status": "success",
            "current_risk_score": last_Y,
            "trend_slope_overall": round(float(slope_all), 6),
            "trend_slope_intervention": round(float(slope_plan), 6),
            "forecast_status_quo": [round(v, 4) for v in forecast_status_quo],
            "forecast_with_plan": [round(v, 4) for v in forecast_with_plan],
            "projected_risk_30d_no_action": round(float(forecast_status_quo[-1]), 4),
            "projected_risk_30d_with_plan": round(float(forecast_with_plan[-1]), 4),
            "retrospective_att": round(att, 4) if att else None,
            "history_days": len(risk_history),
            "intervention_days": int(intervention_mask.sum())
        }

    @staticmethod
    def _weighted_fit(x, y, w):
        W = np.diag(w)
        X = np.vstack([x, np.ones_like(x)]).T
        beta = np.linalg.inv(X.T @ W @ X) @ X.T @ W @ y
        return float(beta[0]), float(beta[1])
```

#### 4.2.2 Level 2 · 群体级跨用户归因 ★

群体归因的核心思路：**找到基线特征相似的用户，比较使用干预的人和没使用干预的人，在同一时间窗口内的健康指标变化。**

```
举例:
  用户A: 身高185, 体重90kg, BMI 26.3, 50岁, 轻度高血压 → 使用 App 干预 90 天
  用户B: 身高183, 体重88kg, BMI 26.3, 52岁, 轻度高血压 → 未使用干预 (对照组)

  → A 和 B 基线几乎一样, 唯一区别是 A 用了我们的干预
  → 比较 A 的血压/风险分变化 vs B 的变化 = 干预因果效应
  → 100 对这样的匹配 → 统计效力远超单人时序
```

**匹配维度**（来自端侧脱敏上传的统计向量）:

| 匹配变量 | 脱敏等级 | 说明 |
|----------|----------|------|
| `age_bracket` | 分段编码 (1-6) | 10岁一档，非精确年龄 |
| `bmi_level` | 分类标签 | underweight/normal/overweight/obese |
| `bp_baseline_grade` | 由 `sbp_mean` 分级 | normal/elevated/hyp_stage1/hyp_stage2 |
| `activity_level` | 由运动热量分级 | sedentary/low/moderate/active |
| `gender` | 二值 | M/F |
| `season_sin`, `season_cos` | 连续值 | 控制季节混杂 |

```python
# bodyup_cloud_actuary/engine/group_attribution.py

import numpy as np
import scipy.optimize as opt
from typing import List, Dict

class GroupAttributionEngine:
    """
    群体级跨用户归因 — 相似用户中干预组 vs 对照组
    用途: 保险公司精算结算的因果效应证据
    """

    MATCHING_FEATURES = [
        "age_bracket", "bmi_level_encoded", "bp_baseline_grade_encoded",
        "activity_level_encoded", "gender_encoded",
        "season_sin", "season_cos"
    ]

    def __init__(self, config: dict):
        self.caliper = config.get("caliper", 0.2)
        self.n_bootstrap = config.get("n_bootstrap", 200)
        self.confidence_level = config.get("confidence_level", 0.95)
        self.min_group_size = config.get("min_group_size", 30)

    def _sigmoid(self, z):
        return 1.0 / (1.0 + np.exp(-np.clip(z, -20, 20)))

    def _compute_propensity(self, X, Z):
        def loss(beta):
            p = self._sigmoid(X @ beta)
            eps = 1e-15
            return (-np.mean(Z * np.log(p + eps) + (1 - Z) * np.log(1 - p + eps))
                    + 0.1 * np.sum(beta ** 2))
        res = opt.minimize(loss, np.zeros(X.shape[1]), method='BFGS',
                           options={'maxiter': 200})
        return np.clip(self._sigmoid(X @ res.x), 0.05, 0.95)

    def _psm_match(self, ps_treated, ps_control):
        """Caliper 最近邻匹配, 返回匹配索引对"""
        matched = []
        used_control = set()
        for i, ps_t in enumerate(ps_treated):
            dists = np.abs(ps_control - ps_t)
            for j in np.argsort(dists):
                if j not in used_control and dists[j] <= self.caliper:
                    matched.append((i, j))
                    used_control.add(j)
                    break
        return matched

    def _fit_outcome(self, X, Y):
        alpha = 1.0
        beta = np.linalg.inv(X.T @ X + alpha * np.eye(X.shape[1])) @ X.T @ Y
        return X @ beta

    def estimate(self, user_records: List[Dict]) -> dict:
        """
        user_records: 每个用户一条记录
        {
            "device_id": str,
            "Z": int,  # 1=使用干预, 0=对照组
            "delta_Y": float,  # 观测期内风险分变化量 (Y_end - Y_start)
            "features": {age_bracket, bmi_level_encoded, ...}
        }
        """
        n = len(user_records)
        if n < self.min_group_size:
            return {"status": "insufficient_users", "n": n,
                    "min_required": self.min_group_size}

        # 构建矩阵
        X_raw = np.array([
            [r["features"].get(f, 0) for f in self.MATCHING_FEATURES]
            for r in user_records
        ])
        X = np.hstack([np.ones((n, 1)), X_raw])
        Z = np.array([r["Z"] for r in user_records], dtype=float)
        delta_Y = np.array([r["delta_Y"] for r in user_records])

        n_treated = int(Z.sum())
        n_control = int((1 - Z).sum())
        if n_treated < 10 or n_control < 10:
            return {"status": "imbalanced_groups",
                    "n_treated": n_treated, "n_control": n_control}

        # Step 1: 倾向得分匹配
        ps = self._compute_propensity(X, Z)
        ps_treated = ps[Z == 1]
        ps_control = ps[Z == 0]
        matched_pairs = self._psm_match(ps_treated, ps_control)

        if len(matched_pairs) < 10:
            return {"status": "insufficient_matches",
                    "n_matched": len(matched_pairs)}

        # Step 2: 匹配后的 DRE
        treated_idx = np.where(Z == 1)[0]
        control_idx = np.where(Z == 0)[0]
        matched_t = np.array([treated_idx[p[0]] for p in matched_pairs])
        matched_c = np.array([control_idx[p[1]] for p in matched_pairs])
        all_matched = np.concatenate([matched_t, matched_c])

        Z_m = Z[all_matched]
        Y_m = delta_Y[all_matched]
        X_m = X[all_matched]

        e_m = self._compute_propensity(X_m, Z_m)
        mu_m = self._fit_outcome(X_m, Y_m)

        mz = np.mean(Z_m)
        dr1 = (Z_m * Y_m) / mz - ((Z_m - e_m) / (1 - e_m)) * (mu_m / mz)
        dr0 = (((1 - Z_m) * e_m * Y_m) / ((1 - e_m) * mz)
               + ((Z_m - e_m) / (1 - e_m)) * (mu_m / mz))
        individual_effects = dr1 - dr0
        att = float(np.mean(individual_effects))

        # Step 3: Bootstrap
        n_m = len(all_matched)
        idx = np.random.choice(n_m, size=(self.n_bootstrap, n_m), replace=True)
        boot_atts = np.mean(individual_effects[idx], axis=1)

        alpha_ci = 1.0 - self.confidence_level
        ci_lo = float(np.percentile(boot_atts, alpha_ci / 2 * 100))
        ci_hi = float(np.percentile(boot_atts, (1 - alpha_ci / 2) * 100))

        return {
            "status": "success",
            "level": "group",
            "att": round(att, 4),
            "ci_lower": round(ci_lo, 4),
            "ci_upper": round(ci_hi, 4),
            "n_total_users": n,
            "n_treated": n_treated,
            "n_control": n_control,
            "n_matched_pairs": len(matched_pairs),
            "n_bootstrap": self.n_bootstrap,
            "is_significant": bool(ci_hi < 0),
            "matching_balance": self._check_balance(
                X_raw, Z, matched_t, matched_c)
        }

    def _check_balance(self, X_raw, Z, idx_t, idx_c) -> dict:
        """匹配后协变量平衡性检查 (SMD < 0.1 为良好)"""
        balance = {}
        for i, feat in enumerate(self.MATCHING_FEATURES):
            t_vals = X_raw[idx_t, i]
            c_vals = X_raw[idx_c, i]
            pooled_std = np.sqrt((np.var(t_vals) + np.var(c_vals)) / 2) + 1e-10
            smd = abs(np.mean(t_vals) - np.mean(c_vals)) / pooled_std
            balance[feat] = {
                "smd": round(float(smd), 4),
                "balanced": bool(smd < 0.1)
            }
        return balance
```

**两层如何协作**：

```
Phase 1 (M1-M6): 
  用户注册 → 端侧记忆文件积累 → 每日脱敏上传
  ↓
  云端:
    CatBoost 每日评分 → Y 值写入用户时序
    Level 1 个人预测 → 用户 App 里看到两条风险走势线 (留存核心)
    数据池持续积累, 等待 100+ 用户
  
Phase 2 (M6+):
  数据池达到 100+ 用户 (含自然对照组)
  ↓
  云端启动:
    Level 2 群体归因 → PSM 匹配相似用户 → DRE 跨用户因果估计
    ↓
    群体级控费报告 → Ed25519 签名 → 提交保司精算师
  
  同时 Level 1 继续运行:
    用户日常体验不变, 每天打开 App 看到个性化预测
```

**对照组从哪来？** 不需要刻意招募"不用 App 的人"：
1. **新注册但尚未开始干预**的用户（注册后到首次执行干预之间的观测期）
2. **低活跃用户**（下载了但很少打开，依从率 < 10%）
3. **等待期用户**（设计随机分步上线，先观测再干预）

**为什么两层都需要**：
- Level 1 解决的是**用户端产品问题**——"这个 App 对我有没有用？" → 用户看到自己的趋势，愿意继续用
- Level 2 解决的是**商业结算问题**——"你凭什么说你降低了医疗支出？" → 精算师看到因果证据，愿意付钱
- 两者数据管线共享（都来自端侧脱敏上传），但分析方法和受众完全不同

### 4.3 Ed25519 非对称签名（替代 HMAC）

```python
# bodyup_cloud_actuary/engine/report_signer.py

from cryptography.hazmat.primitives.asymmetric.ed25519 import (
    Ed25519PrivateKey, Ed25519PublicKey
)
from cryptography.hazmat.primitives import serialization
import json, time, base64

class ReportSigner:
    """
    Ed25519 非对称签名
    vs HMAC: 保险公司拿到公钥只能验签, 不能伪造 → 不可抵赖
    """

    def __init__(self, private_key: Ed25519PrivateKey):
        self.private_key = private_key

    def sign_report(self, attribution_result: dict, device_id: str,
                    disease_type: str) -> dict:
        payload = {
            "device_id": device_id,
            "disease_type": disease_type,
            "timestamp": int(time.time()),
            **attribution_result
        }

        canonical = json.dumps(payload, sort_keys=True,
                               separators=(',', ':'),
                               ensure_ascii=False).encode('utf-8')

        signature = self.private_key.sign(canonical)

        return {
            "version": "1.0.0",
            "signature_algorithm": "Ed25519",
            "payload": payload,
            "signature": base64.b64encode(signature).decode('ascii')
        }

    @staticmethod
    def verify(report: dict, public_key: Ed25519PublicKey) -> bool:
        try:
            payload = report["payload"]
            sig = base64.b64decode(report["signature"])
            canonical = json.dumps(payload, sort_keys=True,
                                   separators=(',', ':'),
                                   ensure_ascii=False).encode('utf-8')
            public_key.verify(sig, canonical)
            return True
        except Exception:
            return False
```

## 6. 技术栈选型

### 端侧

| 层级 | 选型 | 理由 |
|------|------|------|
| 框架 | Flutter | 跨平台，快速交付 |
| 本地存储 | SQLite + AES-256-GCM | 原始数据加密 |
| 特征计算 | C++ (Eigen) / Python (NumPy) | 轻量统计算子 |
| 手表对接 | HealthKit / Samsung Health / Google Fit | 覆盖主流设备 |
| 后台任务 | iOS BGTaskScheduler / Android WorkManager | 做梦机制 |

### 云端

| 层级 | 选型 | 理由 |
|------|------|------|
| API 框架 | FastAPI (Python 3.11+) | 异步高性能 |
| 风险评分 | CatBoost (已训练 AUC=0.847) | 原生分类特征支持, SHAP 可解释, 可扩展多模型 |
| LLM 推理 | Claude API / GPT API / OpenRouter | 零运维按量付费, 后期可迁移 vLLM+Gemma 自建 |
| 归因引擎 | NumPy + SciPy (手写) | 零重型依赖 |
| 签名 | Ed25519 (cryptography 库) | 非对称不可抵赖 |
| 数据库 | PostgreSQL + TimescaleDB | 脱敏时序 |
| 密钥管理 | AWS CloudHSM / Azure Dedicated HSM | 硬件安全 |
| 部署 | Docker + K8s | 微服务编排 |
