# 工程状态

更新时间：2026-07-08

## Android 客户端

当前工程是一个 Kotlin + Jetpack Compose Android 应用，包名为 `com.rehealth.genie`。

已完成的主要流程：

- 启动页
- 登录页
- 健康初识采访页
- 智能戒指绑定页
- 首页
- 健康数据页
- 归因页、模型页、我的页的基础入口

## 智能戒指接入

已接入 MRD 戒指 SDK：

- SDK 文件：`app/libs/sdk_mrd2026_1.3.0.aar`
- BLE 仓库实现：`MrdBleRingRepository`
- SDK 协议适配：`MrdProtocolAdapter`

当前真机验证情况：

- 心率：可测量并保存
- 血氧：可测量并保存
- 血压：可测量并保存
- 体温：通过 SDK 的定时体温检测开启后读取历史数据
- 睡眠：按 SDK 历史数据读取路径接入
- 步数：按 SDK 历史数据读取路径接入

体温不是实时单次测量。SDK 示例中体温主要通过 `setTimingTempTest(true, 30)` 开启定时检测，再读取 `TempHistory`。

## 自动采集

当前已实现 APP 前台自动采集：

- 进入主页后自动启动采集循环
- 自动读取历史数据
- 自动触发心率、血氧、血压测量
- 自动开启体温定时采集并读取历史体温
- 自动重连已知戒指地址，减少用户手动操作

当前间隔：

- `AUTO_COLLECTION_INTERVAL_MS = 15 * 60 * 1000L`

后续如需锁屏和后台长期采集，需要实现 Android 前台服务，并展示系统常驻通知。

## 数据存储

当前使用 Room 本地数据库保存：

- 生命体征测量数据
- 睡眠数据
- 活动数据
- 原始信号片段

服务器尚未接入，当前数据不会上传云端。
