## 架构亮点

### 1. 401检测与队列暂停
```kotlin
// API调用 → 401检测
AuthenticatedApiClient.evaluateFeatures()
  → HTTP 401响应
  → authState = Unauthorized
  → syncRepository.pauseQueue()
  → return ApiResult.Unauthorized

// 重新登录 → 恢复队列
sessionStore.token = jwt
authenticatedApiClient.onLoginSuccess(jwt)
syncRepository.resumeQueue()
```

### 2. 本地优先反馈
```kotlin
// 用户提交反馈 → 永不丢失
interventionFeedbackRepo.submitFeedback(interventionId, status, note)
  → 保存到 intervention_feedback_queue (uploadStatus = "pending")
  → MeasurementSyncWorker.triggerImmediate()
  → Worker后台上传
    → 成功: uploadStatus = "done"
    → 401: 暂停队列
    → 网络错误: 指数退避重试
```

### 3. 周期性Worker
```kotlin
// 每30分钟
MeasurementSyncWorker.doWork()
  1. 检查 syncRepository.canUpload() → false则跳过
  2. 获取 feedbackRepo.getPendingUploads()
  3. 每个item:
     a. uploadFeedback(item)
     b. 如果null → 401，停止worker
     c. 如果updated → 保存状态
  4. 清理旧项
```
